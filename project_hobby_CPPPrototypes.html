C/C++ Prototypes

All these prototypes use minimal amount of external code, that I did not write. The idea to start doing this came from HandmadeHero by Casey Muratori, who is creating a game in C/C++ from scratch with no external libraries, on stream, in a very long running series.



Image Viewer

As you might have noticed, the new standard image viewer on Windows 10 is awful. It is buggy, slow and has problems displaying images with transparency. That is why I had the idea to create my own Image Viewer, with interesting features like a "Folder Overview" and other stuff.
The reason I stopped working on it, was the fact that the amount of work to only load most existing image types is insane. That was highly demotivating and the reason I stopped. But I found a replacement project, the Audio Player, which does not have this problem.

Hex map builder

I am a fan of Pen'n Paper games like Dungeons and Dragons and wanted to make a cool and simple map builder for it. I took the liberty to use hex spaces instead of the normally used squares, because they have much more consistent properties with what you normally want to do in PnP games. Measuring distance for instance. This turorial is the quintessential reference, in my opinion, for hex field stuff.
It quickly became more than just a map builder, as I started to implement path finding and other stuff. Thus I stared some undefined PnP like game.
The reason I stopped was that I had the idea to implement rigid body physics for rolling dice. I stopped when I had some bug in the EPA/GJK algorithm I couldn't find and was busy with other stuff. Like that it faded out of my conscious mind and I stopped working on it.
